# Практическая работа по JUnit

Это практическая задача к модулю «Написание и поддержка тестов на JUnit 5».

В этом проекте уже реализована основная бизнес-логика, но JUnit 5 ещё не подключён. Ваша задача — изучить существующий код, подключить JUnit 5 к Maven-проекту, переписать несколько legacy-тестов с JUnit 4 на JUnit 5 и написать новые тесты для существующего функционала.

## Что есть в проекте

В проекте находятся следующие классы:

- `Main` — вспомогательный класс для запуска программы;
- `OrderCalculator` — основной класс с бизнес-логикой;
- `OrderCalculatorLegacyTest` — legacy-тесты на JUnit 4, которые нужно переписать на JUnit 5.

## Текущий функционал

Класс `OrderCalculator` поддерживает следующие операции:

### 1. Расчёт общей стоимости заказа

Метод:
`calculateTotalPrice(int price, int quantity)`

Что делает:
- рассчитывает итоговую стоимость заказа;
- выбрасывает исключение, если цена отрицательная;
- выбрасывает исключение, если количество меньше или равно нулю.

### 2. Проверка бесплатной доставки

Метод:
`isFreeDelivery(int totalPrice)`

Что делает:
- возвращает `true`, если сумма заказа больше или равна 3000;
- возвращает `false`, если сумма заказа меньше 3000.

### 3. Расчёт скидки по промокоду

Метод:
`calculateDiscountPercent(String promoCode)`

Что делает:
- возвращает скидку `10` для промокода `NEW10`;
- возвращает скидку `15` для промокода `SALE15`;
- возвращает скидку `20` для промокода `VIP20`;
- выбрасывает исключение, если промокод пустой, `null` или неизвестный.

### 4. Определение типа доставки

Метод:
`getDeliveryType(int totalPrice, boolean expressDelivery)`

Что делает:
- возвращает `EXPRESS`, если выбрана экспресс-доставка;
- возвращает `FREE`, если сумма заказа больше или равна 3000 и экспресс-доставка не выбрана;
- возвращает `PAID` во всех остальных случаях.

### 5. Формирование краткой сводки заказа

Метод:
`createOrderSummary(String clientName, int totalPrice, boolean freeDelivery)`

Что делает:
- формирует строку с именем клиента;
- добавляет сумму заказа;
- добавляет информацию о том, бесплатная доставка или платная.

## Что нужно сделать

1. Подключите JUnit 5 к Maven-проекту.
2. Изучите класс `OrderCalculator` и разберитесь, как работает его текущая бизнес-логика.

### Переписать legacy-тесты с JUnit 4 на JUnit 5

В проекте уже есть два теста на JUnit 4. Их нужно переписать на JUnit 5:

1. `calculateTotalPriceShouldReturnCorrectTotalPrice`
2. `calculateDiscountPercentShouldThrowExceptionForUnknownPromoCode`

### Написать новые тесты на JUnit 5

После подключения JUnit 5 и переписывания legacy-тестов нужно дописать новые тесты для класса `OrderCalculator`.

Нужно реализовать:

1. Тест, который проверяет, что метод расчёта общей стоимости заказа выбрасывает исключение, если количество товаров равно нулю.
2. Тест, который проверяет, что метод расчёта общей стоимости заказа выбрасывает исключение, если цена товара отрицательная.
3. Параметризованный тест для проверки метода, который определяет, будет ли доставка бесплатной в зависимости от суммы заказа.
4. Параметризованный тест для проверки метода расчёта скидки по известным промокодам.
5. Параметризованный тест для проверки метода, который определяет тип доставки в зависимости от суммы заказа и выбранного способа доставки.
6. Тест, который проверяет корректность формирования сводки заказа и использует `assertAll()` для объединения нескольких проверок в одном тестовом методе.

## Требования к решению

В решении нужно использовать возможности JUnit 5, которые вы изучили в модуле:

- `@Test`
- `@BeforeEach`
- `@ParameterizedTest`
- `@CsvSource`
- `assertEquals()`
- `assertTrue()`
- `assertFalse()`
- `assertThrows()`
- `assertAll()`

## Ожидаемый результат

После выполнения практической работы в проекте должны быть:

- подключён JUnit 5 в Maven-проекте;
- переписанные тесты с JUnit 4 на JUnit 5;
- новые тесты на JUnit 5;
- параметризованные тесты для проверки бесплатной доставки, расчёта скидки по промокоду и определения типа доставки;
- тест с использованием `assertAll()`;
- успешно проходящий запуск всех тестов.

3. Убедитесь, что все тесты проходят успешно.
4. Проверьте, что legacy-тесты больше не используют JUnit 4.
5. Убедитесь, что новые тесты написаны именно на JUnit 5.
